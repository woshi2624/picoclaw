# 📈 期货交易标准化规则模板生成器

自动拉取 AKShare 国内期货日线数据，计算 20 日均线、前 20 日支撑/压力位、ATR(14)，并额外给出“是否突破区间”“是否处于上涨通道/下跌通道/震荡区间”的判断依据。适合让 AI 直接引用这些字段做分析，而不是只看一段自然语言结论。

## 输出约束

使用这个 skill 时，AI 的最终输出必须严格按照本 skill 生成的模板结构来写，不允许改成自由散文、随意列表或省略关键字段。

如果脚本已经生成了完整模板，AI 应直接复用该结构，并在字段内补充解释；不要重新发明一套格式。

最终输出至少必须按以下顺序包含这些项目：

1. `交易类型`
2. `突破判断`
3. `通道判断`
4. `通道细节`
5. `核心逻辑`
6. `确认信号`
7. `入场价`
8. `止损价`
9. `目标价`
10. `仓位管理`
11. `关键位`
12. `趋势数据`

如果某一项当前无法可靠判断，也必须保留该字段，并明确写 `待确认`、`等待突破/回踩后再定` 或 `N/A`，不能直接省略。

推荐输出骨架如下：

```markdown

| 项目 | 结果 |
| :--- | :--- |

| **2. 突破判断｜是否突破区间** | **...** |
| **3. 通道判断｜上涨 / 下跌 / 震荡** | **...** |
| **4. 通道细节｜高低点斜率** | **...** |
| **5. 核心逻辑｜支撑压力 / 新消息 / 联动品种** | **...** |
| **6. 确认信号｜K线形态 / 关键指标** | **...** |
| **7. 入场价｜执行价格** | **...** |
| **8. 止损价｜认赔离场位** | **...** |
| **9. 目标价｜预期止盈位** | **...** |
| **10. 仓位管理｜投入手数** | **...** |
```

除非用户明确要求别的格式，否则默认必须使用上面的表格模板输出。

除非用户明确要求额外解释、补充分析或改格式，否则 AI 不得在模板前后追加任何自由文本、总结段落、免责声明、额外标题或列表。默认输出应仅包含该模板本身。

## 依赖安装

首次使用前请确保你已经安装了 `uv` 并且可以使用它运行。

## 快速开始

```bash
# 生成螺纹钢（主力）交易计划模板
uv run --with akshare --with pandas scripts/generate_rule.py --symbol RB0

# 生成黄金（主力）交易计划模板
uv run --with akshare --with pandas scripts/generate_rule.py --symbol AU0

# 生成玻璃（主力）交易计划模板
uv run --with akshare --with pandas scripts/generate_rule.py --symbol FG0
```

## 参数说明

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `--symbol` | 标的代码（如 `RB0`, `AU0`, `TA0`） | 必填 |

## 模板规则依据 必须严格按照以下格式输出 

1. **关键位**：自动找出前 20 日最高点（压力）和最低点（支撑），不把当天 K 线算进去。
2. **突破判断**：检查最新收盘价是否已经有效突破前 20 日压力，或跌破前 20 日支撑。
3. **通道判断**：计算近 20 日高点斜率和低点斜率。高低点同步抬升定义为 `上涨通道`，同步下移定义为 `下跌通道`，其余定义为 `震荡区间`。
4. **交易类型**：综合价格相对 MA20 的位置、MA20 斜率，以及是否突破前 20 日区间，区分 `趋势`、`突破` 或 `等待`。
5. **入场价**：默认给出最新收盘价作为参考价，但要求结合回踩/反抽确认后再执行。
6. **止损**：优先使用结构位失效点，再叠加 `0.5 * ATR` 缓冲，同时保证止损距离不小于 `1.5 * ATR`。
7. **目标价**：优先参考前高/前低，或至少保持名义上不低于 `2:1` 的盈亏比。

> 请注意，工具生成的突破判断、通道判断和建议止损只是参考，必须结合实际盘面的 K 线形态、成交量、合约乘数与保证金要求后方可入场。
